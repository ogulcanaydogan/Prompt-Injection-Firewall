# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in PIF, please report it responsibly:

1. **Do not** open a public issue
2. Report via [GitHub Security Advisories](https://github.com/ogulcanaydogan/Prompt-Injection-Firewall/security/advisories/new)
3. Include a detailed description and reproduction steps
4. Allow up to 72 hours for an initial response

## Scope

The following are in scope for security reports:

- Bypass of prompt injection detection
- False negatives in detection rules
- Vulnerabilities in the proxy middleware
- Information disclosure through the API
- Denial of service vulnerabilities

## Recognition

We appreciate responsible disclosure and will credit reporters (with permission) in release notes.
